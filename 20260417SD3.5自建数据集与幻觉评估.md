# SD3.5 自建数据集生成与幻觉评估技术文档

## 1. 项目目标

本项目旨在构建一套基于 **Stable Diffusion 3.5（SD3.5）** 的自建图像数据集，并在此基础上评估多模态模型（当前主要使用 **GPT-4o**）的**对象幻觉（object hallucination）**行为。

核心研究目标包括：

1. 通过 gender-swap prompt 生成受控图像，构建可比较的男性 / 女性主体样本；
2. 在图像中注入 **male_objects** 或 **female_objects**，构造性别偏向物体干预；
3. 使用 GPT-4o 对图像生成描述；
4. 基于多层 GT（`injected / core / extended`）评估模型：
   - 漏掉了哪些真实物体；
   - 错说了哪些物体；
   - 额外幻觉出了哪些物体；
5. 分析：
   - 不同场景是否影响幻觉率；
   - male_objects / female_objects 是否引入系统性差异；
   - `scene × object_condition × prompt_gender` 是否存在交互效应。

## 3. 数据构建逻辑

---

### 3.1 基础 gender-swap 数据集

首先基于 `gender_swap_prompts_en_nobrackets.json` 构建基础图像，保证：

* 场景固定；
* 职业 / 动作 /构图尽可能一致；
* 仅改变主体 gender；
* 同组共享 seed，以减少无关因素扰动。

典型场景包括：

* bar
* board meeting
* home kitchen
* hospital ward
* mining site
* office
* operating room
* school
* soccer field

---

### 3.2 注入物体数据集

然后基于：

```text
/root/Generate_images/gender_swap_prompts_en_objects_mf_singlelib_aggressive.json
```

构造带有显式性别偏向物体的图像。

当前设计为：

* `male_objects`：向图中注入男性偏向物体库
* `female_objects`：向图中注入女性偏向物体库

同时保留：

* `prompt_gender = male`
* `prompt_gender = female`

从而构成 4 类组合：

1. `female_objects + female`
2. `female_objects + male`
3. `male_objects + female`
4. `male_objects + male`

这使得后续可以分析：

* 注入物体本身的主效应
* prompt gender 的主效应
* `scene × object_condition × prompt_gender` 的交互

## 4. 幻觉评估代码文件功能说明

---

### 4.1 `build_gt_manifest.py`

路径：

```text
/root/custom_dataset_eval/hallucination/build_gt_manifest.py
```

功能：

1. 读取注入版 prompt JSON：

   ```text
   /root/Generate_images/gender_swap_prompts_en_objects_mf_singlelib_aggressive.json
   ```
2. 为每个样本构造标准化 GT；
3. 生成 manifest 文件，供评估脚本使用。

#### 输出 GT 分为三层：

### (1) `injected_objects_gt`

仅包含本次人为注入的偏向物体。

作用：

* 评估模型是否真的“看到了”注入物体；
* 判断 injected objects 被模型提及的比例。

---

### (2) `core_gt_objects`

严格版 GT，只保留最核心、最应被提到的对象。

特点：

* 更严格；
* 更容易判出 hallucination；
* 更适合看“真正的错说”。

---

### (3) `extended_gt_objects`

宽松版 GT，除核心对象外，还补充合理背景对象。

特点：

* 更公平；
* 更适合区分“真实幻觉”与“合理补充”。

---

#### 输出文件

通常包括：

```text
/root/custom_dataset_eval/hallucination/results/gt_manifest_singlelib_aggressive/gt_manifest.jsonl
/root/custom_dataset_eval/hallucination/results/gt_manifest_singlelib_aggressive/gt_manifest.csv
/root/custom_dataset_eval/hallucination/results/gt_manifest_singlelib_aggressive/gt_manifest.xlsx
/root/custom_dataset_eval/hallucination/results/gt_manifest_singlelib_aggressive/build_summary.json
```

---

### 4.2 `evaluate_object_hallucination.py`

路径：

```text
/root/custom_dataset_eval/hallucination/evaluate_object_hallucination.py
```

功能：

1. 读取图像目录与 `gt_manifest.jsonl`
2. 自动匹配图像与 manifest 记录
3. 调用 GPT-4o 对图像生成描述

## 5. 幻觉评估指标说明

本项目使用三组评估口径：

1. `injected`
2. `core`
3. `extended`

---

### 5.1 injected 指标

用于评价模型是否把**注入物体**真正说出来。

#### `injected_object_recall`

```text
matched_injected_objects / injected_objects_gt
```

含义：

* 注入物体中，有多少被模型提到了。

#### `injected_object_precision`

```text
matched_injected_objects / predicted_objects
```

含义：

* 模型说出的对象里，有多少是真正属于 injected set。

#### `avg_injected_object_mention_count`

含义：

* 平均每张图提到多少个 injected objects。

---

### 5.2 core / extended 通用指标

#### `CHAIRi_like`

对象级幻觉率：

```text
hallucinated_objects / predicted_objects
```

含义：

* 模型说出的对象里，有多少是 GT 不支持的。

#### `CHAIRs_like`

样本级幻觉率：

```text
至少出现一个 hallucinated object 的样本比例
```

含义：

* 有多少图像至少出现一次 hallucination。

#### `object_precision`

```text
matched_objects / predicted_objects
```

含义：

* 模型说出的对象中，有多少真的被 GT 支持。

#### `object_recall`

```text
matched_objects / gt_objects
```

含义：

* GT 中应出现的对象，有多少被模型说出来。

#### `object_f1`

precision 与 recall 的综合指标。

#### `missing_rate`

```text
missed_gt_objects / gt_objects
```

含义：

* GT 中真实存在的对象，有多少被漏掉了。

基于当前输出，可以回答：

## 9. 当前阶段已经观察到的主要结论方向

根据已有结果，通常会看到如下几类趋势：

1. **场景是强变量**

   * office / soccer field 往往更容易让 injected objects 被提到
   * hospital ward / home kitchen 往往更弱

2. **female_objects 往往比 male_objects 更容易被 GPT-4o 说出来**

   * recall 与 precision 往往更高

3. **prompt_gender 的主效应相对较弱**

   * 全局上 male / female prompt 差异通常小于 scene 或 object_condition

4. **scene × object_condition 可能存在交互**

   * 例如某些场景中 male_objects 更难被模型稳定吸收到描述中

---

## 10. 当前项目的研究意义

这套 SD3.5 自建数据集与 hallucination evaluator 的价值在于：

* 它不是被动使用公共数据，而是可以**主动控制图像内容**；
* 可以精确操控：

  * 主体 gender
  * scene
  * injected object library
* 这使得研究者能够更细粒度地分析：

  * 幻觉是否受到性别偏向物体影响；
  * 不同环境是否会放大这种影响；
  * 模型是否会对某类“性别化物体”更敏感。
