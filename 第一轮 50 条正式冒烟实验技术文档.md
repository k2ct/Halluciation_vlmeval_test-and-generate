````markdown
## 1. 文档目标

本文档用于完整记录第一轮 **50 条正式冒烟实验** 的执行流程。  
本轮实验的目标是：

1. 从公开数据集构建一个**平衡的小规模正式实验集**
2. 将其导出为 **VLMEvalKit 可直接读取的 TSV**
3. 使用 **GPT-4o** 在 AutoDL 上完成图像描述生成
4. 计算初步的 object-level 和 sample-level 幻觉率
5. 检验整条正式实验链路是否已经可用于后续大规模科研实验

---

## 2. 本轮实验的核心思路

本轮实验不是最小 3 条样本测试，而是第一次进入 **正式实验形态**。

实验链路如下：

```text
COCO2017 + Visual Genome
→ 自动筛选候选样本
→ 按 source × scene 平衡采样
→ 导出 VLMEvalKit TSV
→ GPT-4o 生成预测文本
→ 幻觉评估脚本计算指标
````

## 5. 第一步：构建候选样本与平衡采样子集

### 5.1 执行命令

```bash
cd /root/dataset_builder

python build_public_subset.py \
  --out-dir ./outputs_public_subset_v1 \
  --max-per-scene-per-source 80 \
  --min-scene-score 1
```

---

### 5.2 这条命令的作用

该命令完成以下工作：

1. 从 **COCO2017** 中读取：

   * instances annotations
   * captions annotations

2. 从 **Visual Genome** 中读取：

   * objects
   * region descriptions
   * relationships

3. 依据场景词典自动筛选：

   * **有人**
   * **场景明确**
   * 场景属于：

     * street
     * office
     * kitchen
     * school
     * hospital

4. 对每个 `source × scene` 组合最多保留 `80` 条样本

---

### 5.3 参数意义

#### `--out-dir ./outputs_public_subset_v1`

指定输出目录，用于保存候选样本、采样结果和统计信息。

输出目录位置：

```text
/root/dataset_builder/outputs_public_subset_v1
```

#### `--max-per-scene-per-source 80`

表示每个 `(source, scene)` 桶最多采样 80 条。

这里 source 包括：

* coco
* vg

scene 包括：

* street
* office
* kitchen
* school
* hospital

所以理论上最多会得到：

```text
2 × 5 × 80 = 800
```

#### `--min-scene-score 1`

表示只要场景命中分数 ≥ 1 就保留。
这是一个相对宽松的筛选阈值，适合第一轮正式实验。

---

### 5.4 实际输出

命令输出：

```text
COCO candidates: 21104
VG candidates: 28511
Candidates: ./outputs_public_subset_v1/candidate_manifest.jsonl
Sampled:    ./outputs_public_subset_v1/sampled_manifest.jsonl
```

说明：

* COCO 筛出了 21104 条候选
* Visual Genome 筛出了 28511 条候选
* 总候选约 49615 条
* 最终平衡采样得到 800 条

---

### 5.5 输出文件说明

#### 候选样本清单

```text
/root/dataset_builder/outputs_public_subset_v1/candidate_manifest.jsonl
```

作用：

## 8. 第四步：从 800 条中构造 50 条正式冒烟集

```bash

import pandas as pd

    df.groupby(['source', 'scene'], group_keys=False)
      .head(5)
```

### 8.2 这条命令的作用


计算方式：

* 2 个 source：

  * vg

* 5 个 scene：

  * street
  * office
  * kitchen
  * school
  * hospital

* 每桶 5 条

所以总数为：

```text
2 × 5 × 5 = 50
```

### 9.2 执行推理命令

```bash
python run.py \
  --data public_subset_extended_50 \
  --model GPT4o \
  --mode infer \
  --api-nproc 1 \
  --work-dir /root/autodl-tmp/outputs
```

---

### 9.3 参数意义

#### `--data public_subset_extended_50`

指定数据集名称。
VLMEvalKit 会自动去：

```text
$LMUData/public_subset_extended_50.tsv
```

也就是：

```text
/root/autodl-tmp/LMUData/public_subset_extended_50.tsv
```

寻找该 TSV 文件。

#### `--model GPT4o`

指定被测模型为 GPT-4o。

#### `--mode infer`

只执行推理，不执行 VLMEvalKit 自带评测逻辑。
因为本项目的幻觉评估使用的是自定义脚本。

#### `--api-nproc 1`

设置 API 并发数为 1。
这是一种保守但更稳的设置，适合第一轮正式测试。

#### `--work-dir /root/autodl-tmp/outputs`

指定所有推理结果的输出目录。

## 14. 本轮实验的核心结论

### 技术层面

本轮 50 条正式冒烟实验已经证明：

```text
COCO/VG 正式子集
→ VLMEvalKit
→ GPT-4o
→ xlsx 输出
→ 自定义 hallucination evaluator
```

这条完整链路已经稳定可用。

### 研究层面

本轮结果表明：

1. 当前 `extended GT` 仍不足以完全覆盖 GPT-4o 的详细输出
2. 需要进一步优化评估策略，例如：

   * 引入 `core vs extended` 对比
   * 限制对象类别
   * 增强同义词归一化
3. 当前高 hallucination rate 不能直接解释为“模型真的大量幻觉”，而应视为：

   * **模型输出丰富**
   * **GT 不完整**
   * **规则偏严格**
     三者共同作用的结果
